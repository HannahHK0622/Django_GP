Model APIs

This serves as documentation for the REST API.

You are seeing this because the GUI for the REST API implementation is at the directory ~/api/. For each category, the API takes a JSON input, and spits out a JSON response. There are the following categories:

  • Pellet
  • Cargo
  • Order
  • Products

To access the API, please visit ~/api/category/. Alternatively, click here to access the root or the above categories to jump to ~/api/category/.

If you need new classes or have any questions, please raise them in the group or DM me.

Non-Model APIs

You will have to install httpie with `pip install httpie --no-user`.

Test the login API with `http POST :8000/api/login/ username=myuser password=mysecret` while the server is running.

The expected behaviour for a successful login is returning a HTTP code of 202 Accepted.

An unsuccessful one will raise ValidationError with context 'Bad Credentials' and HTTP code 400 Bad Request.

The logout API is tested by `http POST :8000/api/logout` and it will return a message "logged out".

To check if the current user is logged in, call `http GET :8000/api/checkauth`

You may check any error pages by running `command > file` and then opening said file.
